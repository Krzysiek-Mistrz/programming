1) Gettery i Settery sluza do skutecznej ochrony danych, szczegolnie podczas tworzenia klas. Dla kazdej zmiennej metoda get zwraca jej wartosc, natomiast metoda set ustawia jej wartosc.
Gettery zaczynaja sie od get, po ktorym nastepuje nazwa zmiennej, z pierwsza litera nazwy zmiennej pisanej z duzej litery.
Setery zas zaczynaja sie od set po ktorym nastepuje nazwa zmiennej, z pierwsza litera nazwy zmiennej pisanej wielka litera.
Przyklad:
public class Vehicle {
  private String color;

  //Getter
  public String getColor() {
    return color;
  }

  //Setter
  public void setColor(String c) {
    this.color = c;
  }
}
Metoda getter zwraca wartosc atrybutu, natomiast metoda setter bierze parametr i przypisuje go do atrybutu.
Instrukcja "this" jest uzywana do odwolania sie do obecnego obiektu. Wlasciwie this.color jest to atrybut color obecnego obiektu.

3) W chwili kiedy nasze metody getter i setter zostaly zdefiniowane mozemy ich uzyc w funckji main.
public static void main(String[] args) {
  Vehicle v1 = new Vehicle();
  v1.setColor("Red");
  System.out.println(v1.getColor());
}
Gettery i Settery pozwalaja nam miec kontrole nad wartosciami. Mozesz dla przykladu dokonac walidacji danej wartosci w setter zanim dokonasz wlasciego przypisania do zmiennej danej wartosci.
Gettery i Settery sa fundamentalne w procesie enkapsulacji blokow, o tym porozmawiamy w kolejnym module.
